# House Rent Management System v2

A modern, fast, and secure full-stack dashboard for property owners (House Lords) to manage their tenants, calculate monthly bills, record meter readings, and track payments. Built entirely from scratch using clean Next.js 14 paradigms.

## Features

- **Multi-tenant System**: Register as a House Lord, and all data (rooms, tenants, payments) is scoped specifically to your account.
- **Secure Authentication**: Iron-session cookie-based authentication with Next.js edge middleware protecting all dashboard routes.
- **Room Management**: Add rooms, flag them if they require a separate water bill, mark floors.
- **Tenant Assignment**: Allocate rooms to tenants. The system prevents removing occupied rooms.
- **Smart Monthly Billing**:
  - Add **Meter Readings** (prev/current) each month to automatically compute electricity bills based on your settings.
  - Set custom **Water Bills** per eligible room per month.
  - The **Bill Generator Gate** forces water-bills to be completed before full rent is generated, preventing errors.
  - **Carry-forward Dues**: Unpaid balances automatically show up next month's bill!
- **Interactive Dashboard**: KPI metrics and interactive Pie Charts.

## Tech Stack

- **Frontend Framework**: Next.js 14+ (App Router)
- **Styling**: Tailwind CSS (Native utilities + Custom Theme + Glassmorphism)
- **Database**: MongoDB (via Mongoose ODM)
- **Data Fetching**: Axios client + SWR for React hooks
- **Authentication**: `iron-session`
- **Charts**: `recharts`
- **Icons**: `lucide-react`

## Environment Configuration

Create a `.env.local` file in the root folder (automatically generated by AI):
```
MONGODB_URI=mongodb://localhost:27017/house-rent-v2
SESSION_SECRET=a_very_long_complex_32_character_secret_key_change_me
```

## Running Locally

1. Install dependencies:
   ```bash
   npm install
   ```
2. Make sure MongoDB is running on your machine (port `27017`).
3. Start the dev server:
   ```bash
   npm run dev
   ```
4. Navigate to `http://localhost:3000/register` to create your first landlord account!

## Architecture

The system uses a strictly layered approach to avoid tying the User Interface directly to Database models:
1. **Models (`models/`)**: Mongoose definitions (always scoped with `houseLordId`).
2. **Services (`services/`)**: The "Brain". Holds the strict logical rules (like blocking duplicate payments, or carrying forward dues).
3. **API Controllers (`app/api/`)**: Thin wrappers around services.
4. **Client Access (`lib/apiClient.ts`)**: Centralizes all frontend API calls using Axios.
5. **UI Pages (`app/(dashboard)/`)**: Beautiful react-components using reusable Tailwind variants.
